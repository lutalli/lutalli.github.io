In $\mathsf{ZFC}$, everything is a set. Nevertheless, we introduce *classes* to make our life easier. A class is an informal, set-like notion that describes a collection of sets satisfying a certain property. Through classes, we can treat formulas like sets.

Classes only play a role like [syntactic sugar](https://en.wikipedia.org/wiki/Syntactic_sugar). Arguments involving classes are ultimately reduced to formulas.

> **DEF-CLS. Class.**{: .codex-h }
>
> {: .no-indent }
> - If $\varphi(x,p)$ is a formula with free variables among $x$ and $p$, then we define the **class**
>
>   $$ C = \{x:\varphi(x,p)\} $$
>
>   to be an object such that
>
>   $$ \forall x :\enspace x\in C \enspace\Leftrightarrow\enspace \varphi(x,p). $$
>
> - Relations and operations on classes, such as
>
>   $$ C=D ,\enspace C\subseteq D ,\enspace C\cup D ,\enspace C\cap D ,\enspace C\setminus D ,\enspace \bigcup C ,\enspace \bigcap C, $$
>
>   are defined similarly to those on sets.
>
> - $C$ is a **subclass** of $D$ if $C\subseteq D$.
>
> - A class that is not a set is called a **proper class**.
